Introduction

Whatsup guys! So to get started, first I’d like to give you an idea of what you’ll be learning from these Unity Packages. We’re very much aware that there is a lot of Unity content on the web and at times this by itself can be quite overwhelming! So our main goal and mission is to gather as much relevant/fundamental information from the net regarding Unity, organize it, and communicate it with you guys so we can help you spend less time searching for the best links or answers on the Web and so you guys can get started on you’re projects ASAP! (Speaking as a 2nd year Computer Science student though, I still figure you’ll be acquainted quite well with Google nonetheless). Anyway, whether you’re new to Unity or have some experience and you’d like to solidify the fundamental knowledge of it, these packages are for you! Also as a heads up, we’ll be adding more valuable content as time passes to help you fulfill your quest for knowledge!

What the heck is Unity?

It’s a game engine, which simply means it’s a software framework that’s designed for the creation and development of Video Games! You’re able to make both 2D and 3D games with it.

What are the basic skills and tools you'll need to develop on Unity?

1) Get Familiar with Unity’s User Interphase
One of the most important basic skills you’ll have to have is a solid understanding of how to use the user interphase for Unity. In other words, much like how you would need to practice and learn Adobe Photoshop’s tools, hotkeys, and the settings provided visually, you’d have to get familiar with the ones provided on the Unity platform instead.

2)Scripting (C# or JavaScript)
The next thing would be scripting! Now by scripting I mean you should have at the least a basic knowledge of programming. In other words, Object-Oriented Programming I and II at Concordia are both going to be really handy courses here. And if you really don’t want to even touch programming, which I highly doubt, but nonetheless is valuable to know, you can purchase a visual scripting tool called Playmaker on the Unity asset store. In other words you’re still programming but you’re using visual representations of your variables, objects, functions, and the connections between all of these things. (More on that later) Now in terms of what language you would need to know to script in Unity, you’d need either C# or JavaScript. Personally, I think C# is the way to go, especially if you already took some Java courses, which is very probable! And if you check out just a few tutorials on C#, I guarantee you you’ll see many similarities between the languages.

3) Git (Version Control System)+ GitHub
Lastly, if you ever plan to work on a project with other people related to Unity, you MUST learn how to use a Version Control System. Now this is not the formal definition, but the basic idea is that you need to use a tool that allows multiple individuals to be working on the same project at the same time, and make it as convenient as possible to slowly integrate all of your team’s work into one final product. For this, I’d recommend Git and Github. Why not just Git or why not just Github? Because you need a local and a central repository. What that means in really simple words, you’ll need a space where you’re working on a version of your project’s code (that’s you’re computer and you’ll be using Git to communicate with GitHub and have strong control over document changes), and a central space where the final draft of you’re groups merged work can be located (i.e. on GitHub). In any case, I’ll be providing you with the necessary download links and relevant information to quickly set you up with all of these tools in the next section to get you started with the basics and skills you’ll need to create. I hope this has given you a clear and general idea about the journey you’re about to embark on! Starting now, we’re going to delve in deeper!
